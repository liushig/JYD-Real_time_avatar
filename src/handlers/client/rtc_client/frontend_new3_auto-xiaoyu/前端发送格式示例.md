### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 固定为 `"chat"` |
| `data` | object | 是 | 消息数据对象 |
| `data.query` | string | 是 | 用户的问题文本 |
| `data.conversation_id` | string | 是 | 会话ID，首次为空字符串`""`，后续使用后端返回的conversation_id |
| `data.inputs` | object | 是 | 输入数据对象 |
| `data.inputs.data` | string | 是 | 病例数据的JSON字符串（需要转义） |
| `data.user` | string | 是 | 用户标识 |

### 注意事项

1. **JSON格式必须正确**：确保所有字段之间有逗号分隔
2. **inputs.data是字符串**：病例数据需要先转为JSON字符串，然后作为inputs.data的值
3. **不需要headers**：Authorization等请求头由后端自动添加
4. **conversation_id**：
   - 首次对话：传空字符串 `""`
   - 后续对话：使用后端返回的 `conversation_id`

## 后端返回格式

### 数据处理流程说明

**重要：后端不会原封不动返回大模型工作流的SSE输出**

大模型工作流API返回的是SSE（Server-Sent Events）流，包含多种事件类型：
- `workflow_started` - 工作流开始
- `node_started` - 节点开始
- `node_finished` - 节点完成
- `message` - 大模型输出的文本内容

**后端处理逻辑**（`llm_handler_zhongyi_streaming.py`）：
1. 只过滤和处理 `event == "message"` 的事件
2. 从message事件中提取 `answer` 字段（文本片段）
3. 对文本进行缓冲和分句处理
4. 将处理后的文本封装为DataBundle对象
5. 通过WebRTC DataChannel发送给前端

**原始SSE格式示例**（后端接收）：
```
data: {"event": "workflow_started", "task_id": "xxx", ...}
data: {"event": "node_started", "node_id": "xxx", ...}
data: {"event": "message", "answer": "你好", "conversation_id": "xxx", ...}
data: {"event": "message", "answer": "，我是", "conversation_id": "xxx", ...}
data: {"event": "node_finished", ...}
```

**前端实际收到的格式**（通过WebRTC DataChannel）：
```json
{
  "type": "chat",
  "message": "你好，我是",
  "id": "xxx",
  "role": "avatar",
  "conversation_id": "0cb930b8-d3fe-469d-a50f-861b3c0086b6",
  "diagnosis_progress": {
    "tyb": 1,
    "tyinb": 0,
    "syb": 0,
    "syinb": 0,
    "hlb": 0,
    "lfb": 0,
    "jyb": 0,
    "ymb": 1
  }
}
```

### 返回字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | string | 消息类型，`"chat"` 或 `"avatar_end"` |
| `message` | string | 数字人回复的文本内容（从SSE的answer字段提取并缓冲） |
| `id` | string | 消息ID |
| `role` | string | 角色，`"human"` 或 `"avatar"` |
| `conversation_id` | string | 会话ID，用于下一轮对话（从SSE提取） |
| `diagnosis_progress` | object | 问诊进度数据（如果有，从SSE提取） |

### 特殊消息类型

**对话结束标志**：
```json
{
  "type": "avatar_end"
}
```
当数字人回复完成后，会发送此消息通知前端对话结束。
